# Open Prompts Hub

Библиотека структурированных промптов с использованием XML-стиля разметки для эффективного взаимодействия с языковыми моделями.

## 📖 О проекте

Open Prompts Hub — это открытая коллекция промптов, построенных на методологии **XML-style разметки**. Данный подход превращает промпт-инжиниринг из искусства в инженерную дисциплину через четкую иерархическую структуру семантических тегов. Полный шаблон структуры описан в [xml-style.xml](xml-style.xml).

## 🏗 Категории промптов

| Категория | Папка | Описание |
|-----------|-------|----------|
| **Data Engineering** | `data-engineering/` | Пайплайн обработки данных: генерация мок-датасетов с ошибками, очистка и обнаружение аномалий, обезличивание PII, аналитическая саммаризация, визуализация, генерация SQL |
| **Human Resources** | `human-resources/` | Подбор и оценка персонала: создание и скоринг резюме под AI/ML позиции, транскрибация и анализ технических интервью |
| **Research** | `research/` | Исследование и структурирование: mind map документов и тем в формате Markmap, глубинный анализ книг и научных статей |
| **Marketing Analytics** | `marketing-analyst/` | Сквозная аналитика: сбор данных из CRM и рекламных каналов, нормализация, сверка, метрики (CAC/LTV/ROMI), алерты, отчёты |

## 🚀 Использование

1. Выберите подходящий промпт из библиотеки
2. Адаптируйте содержимое тегов под вашу задачу
3. Используйте промпт в любом LLM-интерфейсе (ChatGPT, Claude, и т.д.)

## 🤝 Как добавить промпт

1. Создайте файл в соответствующей категории
2. Используйте единую XML-структуру из [шаблона](xml-style.xml)
3. Добавьте понятное описание цели и контекст
4. Следуйте принципам:
   - Разделяйте команды и контекст
   - Используйте вложенные теги для сложных структур
   - Дробите монолитные инструкции на под-инструкции
   - Называйте теги семантически (snake_case или kebab-case)

## 📚 Рекомендации

- Создавайте модульные, переиспользуемые блоки промптов
- Генерируйте черновики промптов с помощью ИИ, затем проводите ревью
- Избегайте семантической путаницы — разделяйте инструкции, контекст и примеры
- Поддерживайте библиотеку протестированных компонентов
