# Open Prompts Hub

Библиотека структурированных промптов с использованием XML-стиля разметки для эффективного взаимодействия с языковыми моделями.

## 📖 О проекте

Open Prompts Hub — это открытая коллекция промптов, построенных на методологии **XML-style разметки**. Данный подход превращает промпт-инжиниринг из искусства в инженерную дисциплину через четкую иерархическую структуру семантических тегов.

### Преимущества подхода

- **Четкие семантические границы** — модель воспринимает промпт как конфигурационный файл
- **Модульность** — создание библиотеки переиспользуемых блоков
- **Ссылочная архитектура** — связная система знаний вместо линейных инструкций
- **Снижение когнитивной нагрузки** — attention матрицы не тратят ресурсы на разгадывание структуры

## 🏗 Структура

Репозиторий организован как коллекция `.xml`-файлов, каждый из которых содержит готовый промпт или компонент для сборки промптов.

### Базовые компоненты

| Компонент | Описание |
|-----------|----------|
| `tools` | Описание доступных инструментов и функций |
| `personality` | Правила поведения, тон и роль агента |
| `context` | Фоновая информация для выполнения задачи |
| `user_profile` | Описание пользователя для персонализации |
| `output_rules` | Правила форматирования результатов |

### Структура промпта

Каждый промпт следует единому шаблону:

```xml
<objective>Цель промпта</objective>
<context>Контекст и ограничения</context>
<user_profile>Профиль пользователя</user_profile>
<instructions>Инструкции и порядок действий</instructions>
<output_format>Формат ответа</output_format>
<examples>Примеры входов/выходов</examples>
<rules>Правила и ограничения</rules>
<tools>Доступные инструменты</tools>
```

## 🚀 Использование

1. Выберите подходящий промпт из библиотеки
2. Адаптируйте содержимое тегов под вашу задачу
3. Используйте промпт в любом LLM-интерфейсе (ChatGPT, Claude, и т.д.)

### Адаптация промпта

```xml
<!-- Замените содержимое тегов под вашу задачу -->
<objective>
    Ваша конкретная цель
</objective>
<context>
    <background>
        Контекст вашей задачи
    </background>
</context>
```

## 🤝 Как добавить промпт

1. Создайте `.xml`-файл в корне репозитория
2. Используйте структуру из [шаблона](xml-style.xml)
3. Добавьте понятное описание цели и контекст
4. Следуйте принципам:
   - Разделяйте команды и контекст
   - Используйте вложенные теги для сложных структур
   - Дробите монолитные инструкции на под-инструкции
   - Называйте теги семантически (snake_case или kebab-case)

## 📚 Рекомендации

- Создавайте модульные, переиспользуемые блоки промптов
- Генерируйте черновики промптов с помощью ИИ, затем проводите ревью
- Избегайте семантической путаницы — разделяйте инструкции, контекст и примеры
- Поддерживайте библиотеку протестированных компонентов
